# Курсовая работа по дисциплине «Технологии программирования»

## Описание проекта
Проект представляет собой консольное приложение, имитирующее процесс покупки товаров и услуг в магазине техники. 
Программа разработана в рамках курсовой работы по дисциплине «Технологии программирования» и демонстрирует применение 
принципов объектно-ориентированного программирования, а также шаблонов (паттернов) проектирования.

## Назначение программы
Программа предназначена для моделирования процесса взаимодействия покупателя с магазином техники. 
Пользователь может просматривать каталог, добавлять товары в корзину, выполнять взвешивание отдельных товаров и оплачивать 
покупки различными способами (наличные, бонусы, смешанная оплата).

## Требования для запуска
Для запуска программы необходимо:
- Python версии 3.10 или выше;
- среда разработки PyCharm (рекомендуется) или любой другой Python-интерпретатор.

## Запуск программы
Запуск выполняется через файл `main.py`, расположенный в папке `src`.

Команда для запуска из терминала:
`python src/main.py`

## Описание пользовательского интерфейса
Программа использует командно-текстовый интерфейс. После запуска отображается главное меню, 
в котором пользователь выбирает действия, вводя номер пункта.

### Главное меню
Доступные пункты меню:
1 — Каталог (просмотр/поиск)  
2 — Добавить в корзину  
3 — Показать корзину  
4 — Взвесить товар в корзине  
5 — Удалить из корзины  
6 — Оплатить  
7 — История покупок  
0 — Выход  

## Работа с каталогом
Пользователь может:
- вывести полный список товаров и услуг;
- выполнить поиск по названию (вводится часть названия, например «ноут»).

## Работа с корзиной
Пользователь может:
- добавлять товары и услуги в корзину;
- просматривать корзину и итоговую сумму;
- удалять позиции из корзины.

Для товаров, которые требуют взвешивания, перед оплатой необходимо указать вес. Если в корзине есть невзвешенные товары, 
программа запросит взвешивание и не позволит перейти к оплате, пока вес не будет указан.

## Оплата покупки
Поддерживаются способы оплаты:
- только наличными средствами;
- только бонусными баллами;
- смешанная оплата (часть бонусами, часть наличными).

Если у покупателя недостаточно средств для оплаты, программа предложит удалить позиции из корзины до тех пор, пока оплата 
не станет возможной.

## История покупок
После успешной оплаты информация о покупке сохраняется и может быть просмотрена пользователем в пункте «История покупок». 
Дополнительно чек сохраняется в файл `receipts.txt`.

## Заключение
Разработанное приложение соответствует требованиям задания, демонстрирует применение ООП и основных подходов к построению программных систем 
и может использоваться в учебных целях.
